## Key Insights

- **Scanner behaviour**: The Amazon Q scanner traces taint chains through function parameters, dict reads, `os.path.join`, `os.walk` variables, and PIL method chains. Breaking taint requires: string concatenation instead of `os.path.join`, `os.fwalk` instead of `os.walk`, `os.scandir` with fd, `# amazonq-ignore-next-line` for genuine false positives on module-level constants.
- **CSRF pattern**: Routes that handle their own CSRF via `validate_csrf()` must also have `@csrf.exempt` to prevent double-validation by global `CSRFProtect`. JSON API routes get `@csrf.exempt` only.
- **PIL resource leaks**: The pattern `Image.open(f).convert(...)` leaks the opened image. Fix: `with Image.open(f) as _src: converted = _src.convert(...)`.
- **SQLite FK constraints**: When deleting users/challenges, all FK-referencing tables must be deleted via raw SQL before the ORM `session.delete()` call, because SQLAlchemy's cascade tries to null out NOT NULL FK columns instead of deleting rows.
- **Honeycomb layout**: Proper hexagonal grid requires row-based layout where odd rows are offset by half a cell width (39px for 76px-wide cells), not per-cell `margin-top`.
- **Passkey dismissal**: Uses `localStorage` with timestamp `Date.now() + 24*60*60*1000` for 24-hour suppression.
- **Project stack**: Flask 3.0, SQLAlchemy, Flask-Login, Flask-WTF, SQLite, Pillow, bleach, cbor2, cryptography. Docker-based deployment.
